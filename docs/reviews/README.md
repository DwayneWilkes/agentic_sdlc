# Review Reports

This directory contains requirements review reports generated by the Requirements Reviewer Agent.

## Purpose

Review reports validate that implementations:

- Meet all requirements from the roadmap
- Pass all quality gates
- Follow project conventions
- Are well-tested and documented

## Report Format

Each review report includes:

1. **Executive Summary** - Quick overview of findings
2. **Requirements Compliance** - What was met/unmet

## File Naming

Reports are named after the work stream they review:

```
phase-1-1-core-data-models.md
phase-1-2-task-parser.md
phase-2-1-team-composition.md
```

## Usage

### For Implementers

After completing a work stream, check for a review report:

```bash
ls docs/reviews/phase-*.md
cat docs/reviews/phase-1-1-core-data-models.md
```

If changes are required, address the findings and request re-review.

### For Reviewers

Generate a review report following the "Deep Code Review" workflow in [`.claude/agents/qa_agent.md`](../../.claude/agents/qa_agent.md).
